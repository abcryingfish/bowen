前端「总买入信号」由什么构成

口径与实盘因子库一致；实现源码：ZXW因子/总买入信号_独立全量.py · 函数 build_total_buy_signal_bundle
宽表列名：total_buy_signal · 可视化映射：可视化/market_data_service.py → 「总买入信号」

一、主干逻辑（三件事同时成立）

先定义「基础三合一」布尔条件 base：

前端常用名 / 口径英文名 / 字段何为「有信号」
MAC总（D 类） mac_total mac_total > 0（0～4 档综合得分里有任意非零档）
KDJ信号（J 超卖） r_condition J < 30（源码与 KDJ bundle 一致）
抄底总分 bottom_fishing_score bottom_fishing_score > 0
base_signal = (mac_total > 0) ∧ (r_condition 为 True) ∧ (bottom_fishing_score > 0)

来源子模块：MACD因子.build_d_class_factor_bundle、KDJ因子.build_kdj_factor_bundle、抄底因子.build_bottom_fishing_factor_bundle。

二、筹码加分（仅在能算筹码时）

当同时具备 成交量矩阵 V 且能加载 筹码结构因子 时，在 base 已为真 的前提下，对中间原始分 raw_total_buy_signal 最多再加 2 分：

条件因子列加分
集中总评分 > 0（筹码集中度综合评分） concentration_total_score · 前端常称 集中总 一脉 在满足 base 的当日 +1
筹码峰评分 > 0（筹码峰赋值） chip_peak_score 在满足 base 的当日再 +1

筹码由 筹码结构因子.build_chip_structure_factor_bundle 计算（需 OHLCV + 换手率等；本 bundle 内调用参数含 window_days=100, grid_size=600, history_decay=0.95）。

三、原始分与最终「总买入信号」

  • 令 raw_total_buy_signal：先由 base 得 1 或未触发得 0；若筹码分支生效，再在 base 为真时按需 +1/+1。
  • 理论上 raw ∈ {0,1,2,3}（无筹码时只靠 base，raw 只会是 0 或 1）。
  • 总买入信号（total_buy_signal）：raw_total_buy_signal ≥ 2 则为 1，否则为 0。
total_buy_signal = 1 ⇔ raw_total_buy_signal ≥ 2
重要边界：若无成交量或筹码 bundle 不可用，筹码两项加分不会发生，此时 raw 最高为 1，则 总买入信号恒为 0。详见仓库内说明页 backtrader/models/总买入信号改_流程图.html 底部注释。

四、与「总买入信号改」「总买入超强底」

输出宽表列规则摘要
总买入信号改 total_buy_signal_adjusted total_buy_signal > 0 且 筹码单峰优成立：single_peak_best > 0（与前端「筹码单峰优」同链路的单峰最佳状态）
总买入超强底 super_strong_bottom 源码侧与「总买入信号改」当前为同一条件（均为总买入信号 ∧ 筹码单峰优）

五、与其它信号的区别（便于对照）

五日内六级 / 通达信超强底走 ZXW因子/通达信强底信号.py：六路 MAC总信号、K线底信号、集中总信号、筹码峰信号、均线类信号、KDJ超卖 R 等与「总买入信号」的 MAC+KDJ+抄底+筹码加权不是同一套合成式。

回测强点模型里前端自选「买入因子」合成的是 strong_buy_signal，再写入宽表列 total_buy_signal 供 feed 对齐（参见 signal_data.merge_strong_buy_signal），语义以所选因子组合为准，与因子库自带的「总买入信号」公式可并存于不同产品线。
