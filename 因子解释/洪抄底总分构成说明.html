前端「洪抄底总分」逻辑

源码：ZXW因子/洪抄底.py · build_bottom_fishing_factor_bundle
输出列：hong_bottom_fishing_score（对内由 bottom_fishing_score 加前缀 hong_）
前端映射：可视化/market_data_service.py · 「洪抄底总分」→ hong_bottom_fishing_score

一、与「抄底总分」(K线底) 的差异

洪抄底总分与主线 抄底总分（抄底因子.py，即 K线底）不是同一套公式：洪版用「多级低点窗口 + 自上次见底以来的反弹幅度」判定，输出为 0、0.5、1、2、3 离散分；不涉及 K线底1/2、S笔反K 等通达信式合成。

二、五类「洪底」触发（当日是否创新低相对窗口）

对低价 L，与上一日各周期最低比较；并要求历史根数足够（避免冷启动假信号）：

内部名条件（简化）最少上市 K 线数
mini_bottom 洪迷你底L ≤ REF(LLV(L,7), 1)≥ 8
small_bottom 洪小底L ≤ REF(LLV(L,20), 1)≥ 21
major_bottom 洪大底L ≤ REF(LLV(L,200), 1)≥ 201
near_historical_bottom 洪近历史大底L ≤ REF(LLV(L,400), 1)≥ 401
historical_bottom 洪历史大底L ≤ REF(LLV(L,1000), 1)≥ 1001

三、「反弹幅度」与有效状态 active

对每一类底，在给定信号序列上：

  • 用 BARSLAST(信号) 定位最近一次该类见底日。
  • 取该日的信号低价 last_low；从该股价段起至高，算 rise = (HHV(H, BARSLAST+1) − last_low) / last_low（自见底以来最高价相对最低价的最大反弹比例）。
  • 仅在「已累计出现过该类信号」的位置上参与后续判定；并要求 rise < 0.20（20%），表示见底后尚未大幅反弹离开低价区，该类记为 *_active。

五类均套用同一反弹阈值：< 20%。

四、互斥优先级与洪抄底总分

在当日格点上，从上到下只做一档（先匹配者不向下覆盖）：

  1. 洪历史大底 active → 3
  2. 否则若 洪近历史大底 active → 2
  3. 否则若 洪大底 active → 1
  4. 否则若 洪小底 active → 1
  5. 否则若 洪迷你底 active → 0.5
  6. 否则 → 0
hong_bottom_fishing_score = 上表分值；且 base_score ≤ 0 时写 0（本实现中未出现负分）

五、默认回看（增量/切片）

get_factor_lookback_config() 声明：整包约需 1100 日级别上下文；hong_bottom_fishing_score 与 hong_historical_bottom 的 lookback 同为 1100（见文件内 FACTOR_LOOKBACK_DAYS）。
